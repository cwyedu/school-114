## 🌐網站的基本檔案架構

同學們在開始寫 HTML、CSS 之前，
一定要先學會一件事：

👉 網站的檔案怎麼放？
👉 資料夾怎麼整理？

如果一開始架構亂掉，之後寫再多程式都會很痛苦。

---

### 一、什麼是「根目錄」？

我們這學期的網站專案資料夾名稱統一為：

114-web

這個資料夾就是整個網站的「根目錄」。

📌 重要觀念：

* 根目錄是網站最外層資料夾
* 所有網頁檔案都放在裡面
* 網站開啟時，會從這裡開始

你可以把根目錄想成：

🏠 一棟房子的最外層
裡面再分不同房間（資料夾）

---

### 二、請建立以下檔案架構

請使用[Visual Studio Code](https://code.visualstudio.com/)，照著下面建立資料夾與檔案：

📂 檔案樹狀圖（Tree 結構）：

```
114-web/
│
├── index.html
├── style.css
│
├── lab1/
│   └── index.html
│
└── lab2/
    └── index.html
```

說明：

* 114-web 是最外層資料夾（根目錄）
* ├── 代表同一層的檔案或資料夾
* └── 代表該層最後一個項目
* 縮排代表「裡面包含」的意思

⚠ 檔名大小寫要完全正確！
⚠ 副檔名一定要是 .html

---

### 三、每個檔案在做什麼？

#### 1️⃣ 根目錄的 index.html

這是網站的「首頁」。

當瀏覽器開啟一個資料夾時，如果沒有指定檔名，
會自動讀取：

index.html

所以幾乎每個網站都會有這個檔案。

首頁通常會放：

* 網站標題
* 課程名稱
* 自我介紹
* 連到 lab1、lab2 的連結

---

#### 2️⃣ style.css

這是網站的樣式檔。

請記住這句話：

HTML = 內容
CSS = 外觀

HTML 負責：

* 文字
* 標題
* 圖片
* 連結

CSS 負責：

* 顏色
* 字體大小
* 背景
* 排版

在 HTML 裡面要加入這一行，才能使用 CSS：

---

#### 3️⃣ lab1、lab2 資料夾

每一次作業都放在自己的資料夾裡面。

例如：

lab1/
index.html

代表：

* 這是第一次作業
* 有自己獨立的網頁

當進入 lab1 資料夾時，瀏覽器會自動打開：lab1/index.html

---

### 四、為什麼要這樣分開？

如果全部檔案都丟在同一層，會發生什麼事？

❌ 檔案混在一起
❌ 不知道哪個是第幾次作業
❌ 容易刪錯檔案
❌ 之後會非常亂

正確做法是：

✔ 每次作業一個資料夾
✔ 首頁放在根目錄
✔ CSS 統一管理

這是業界也會使用的做法。

---

### 五、本章一定要會的觀念

請確認你可以回答下面問題：

1️⃣ 什麼是根目錄？
2️⃣ 為什麼網站通常都有 index.html？
3️⃣ HTML 和 CSS 的差別是什麼？
4️⃣ lab1 資料夾裡為什麼也要有 index.html？

如果答不出來，請回去再看一次。

---

### 六、課堂實作步驟

請實際操作一次：

1. 建立 114-web 資料夾
2. 建立 index.html
3. 建立 style.css
4. 建立 lab1 資料夾
5. 在 lab1 裡建立 index.html
6. 用瀏覽器打開測試

做到這裡，你已經完成：

🎉 第一個網站的基本架構

### 七、先用 AI 寫個首頁（學會下指令）

現在我們試試看用 AI 幫我們產生首頁程式碼。

⚠ 重點不是抄答案，而是學會「怎麼下指令」。

一個好的 Prompt（指令）要寫清楚：

* 我要什麼語言？（HTML / CSS）
* 要做什麼頁面？（首頁）
* 需要什麼內容？（標題、自我介紹、連結）
* 要不要加樣式？

---

#### 範例 Prompt

請幫我寫一個 HTML 首頁程式碼。

需求：

1. 有一個大標題：我的個人網站
2. 有一段自我介紹文字
3. 有兩個連結，分別連到 lab1 和 lab2
4. 使用基本的 CSS，讓背景顏色是淺灰色，標題置中
5. 請給我完整的 HTML 程式碼

---

#### 進階一點的 Prompt

請幫我寫一個適合高中生作業的簡單首頁。

* 使用 HTML5 基本結構
* 使用外部 style.css
* 版面簡單乾淨
* 不要使用太複雜的語法

---

📌 上課規則：

✔ 可以用 AI 幫忙
✔ 但一定要看懂程式碼
✔ 學會下指令，比抄程式碼更重要。

## 🚀上傳 GitHub（使用 GitHub Desktop）

當網站完成後，我們要把它上傳到網路上保存。

這學期我們統一使用：

👉 GitHub Desktop（圖形化操作，比打指令簡單）

---

### 一、第一次上傳專案流程

請依照以下步驟操作：

① 開啟 GitHub Desktop

② 點選「Add an Existing Repository from your Hard Drive」

③ 選擇你的 114-web 資料夾

④ 如果出現「This directory does not appear to be a Git repository」
請點擊「create a repository」

⑤ Repository name 確認為：114-web
Local path 確認是你的 114-web 資料夾
然後點擊 Create Repository

⑥ 在 Summary 輸入：first commit
點擊 Commit to main

⑦ 點擊右上角「Publish repository」

⑧ 確認：

* Name：114-web
* 設為 Public

然後點擊 Publish Repository

完成後，打開 GitHub 網站，就會看到你的專案。

---

### 二、基本介面與設定

#### GitHub 基本介面認識

進入 Repository 之後，你會看到：

* Code（檔案區）
* Issues
* Settings（設定）

平常最常用：

📁 Code → 看檔案
⚙ Settings → 設定網站

---

#### 啟用 GitHub Pages

GitHub Pages 可以把你的網站變成真正的網址。

設定步驟：

1. 進入 Repository
2. 點選 Settings
3. 點選 Pages
4. Branch 選擇 main
5. 點擊 Save

幾分鐘後會出現一個網址，例如：

[https://你的帳號.github.io/114-web/](https://你的帳號.github.io/114-web/)

這就是你的網站網址。

---

### 三、下次上課流程：下載後再上傳

從第二次上課開始，不是重新建立專案。

正確流程是：

👉 先下載（Clone）
👉 修改作業
👉 再上傳（Push）

這才是正常的開發流程。

---

#### 一、先下載專案（Clone）

① 打開 GitHub Desktop

② 點選 File → Clone Repository

③ 選擇你自己的 114-web

④ 選擇要存放的資料夾位置

⑤ 點擊 Clone

完成後，專案就會出現在你的電腦裡。

---

#### 二、修改作業

1. 打開 114-web 資料夾
2. 進入 lab2（或新的作業資料夾）
3. 修改或新增檔案
4. 存檔

---

#### 三、再上傳更新（Commit + Push）

① 回到 GitHub Desktop

② 在 Summary 輸入本次作業內容（例如：add lab2）

③ 點擊 Commit to main

④ 點擊 Push origin

完成後，GitHub 網站上的內容就會更新。

---

這才是之後每一堂課的標準流程：

下載 → 修改 → Commit → Push

請不要每次都重新建立 Repository。

---

到這裡，你已經完成：

✔ 建立網站架構
✔ 使用 GitHub Desktop 上傳專案
✔ 發布成真正網址
✔ 使用 GitHub Desktop 下載專案

這就是一個完整的網站開發基本流程（入門版）。
