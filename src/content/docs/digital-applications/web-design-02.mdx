使用 AI，製做一個抽籤網頁。本次練習的目標：

1. 學會使用 **GitHub Desktop** 下載 (clone) 專案
2. 了解 **GitHub Pages 網站結構**
3. 建立一個 **首頁 (root index.html)** 作為各 Lab 的入口
4. 使用 **AI 協助產生程式** 製作一個抽籤遊戲
5. 學會修改後 **commit 與 push 到 GitHub**

重點不是自己從零寫程式，而是 **學會如何正確使用 AI 協助開發**。

---

## 一、專案目錄結構

上次上傳課本課程的網站結構如下：

```
114-web/
│
├─ index.html
├─ style.css
│
├─ lab1/
│   ├─ index.html
│   └─ style.css
│
├─ lab2/
│   ├─ index.html
│   └─ style.css
│
└─ lab3/
    ├─ index.html
    └─ style.css
```

說明：

* `114-web` 是 **網站根目錄**
* `index.html` 是 **首頁**
* `lab1 / lab2 / lab3` 是不同的專案
* 每個 lab 都有自己的 `index.html`

首頁的作用是 **提供連結，進入各 lab**。

---

## 二、使用 GitHub Desktop Clone 專案

#### 步驟 1

開啟 **GitHub Desktop**。

如果尚未登入 GitHub，先登入帳號。

---

#### 步驟 2

點擊：

```
File → Clone repository
```

---

#### 步驟 3

在 repository 清單中找到你的專案。

選擇電腦儲存位置後，按下：

```
Clone
```

---

#### 步驟 4

Clone 完成後：

點擊

```
Show in Explorer
```

用 **VS Code 或其他編輯器**開啟資料夾。

---

## 三、建立網站首頁（使用 AI 產生）

這一部分 **不要自己手寫程式**，而是練習 **如何用 AI 幫你建立首頁**。

首頁的目的只有一個：

**提供連結到各個 Lab 的 index.html**。

例如首頁應該可以連到：

* lab1
* lab2
* lab3

---

### Step 1：請 AI 幫你產生首頁

可以向 AI 描述需求，例如：

```
請幫我做一個簡單的網站首頁

需求：
- 使用 HTML
- 顯示標題 "114 Web Labs"
- 有三個連結
- 連到 lab1、lab2、lab3
- 適合初學者
```

AI 會產生一個首頁範例。

---

### Step 2：閱讀 AI 產生的程式

不要直接複製貼上。

先看懂：

* 哪一行是 **標題**
* 哪裡是 **連結 (link)**
* 連結是否真的指向

```
lab1
lab2
lab3
```

---

### Step 3：建立根目錄首頁

把 AI 產生的內容放到：

```
114-web/index.html
```

如果 AI 有產生 CSS，則放到：

```
114-web/style.css
```

---

### Step 4：測試首頁

直接在電腦開啟：

```
index.html
```

確認：

* 首頁可以正常顯示
* 點擊 Lab1 可以進入 lab1
* 點擊 Lab2 可以進入 lab2
* 點擊 Lab3 可以進入 lab3

如果連結錯誤，可以再請 AI 協助修改。

---

## 四、Lab1 任務：AI 抽籤遊戲

在 `lab1` 目錄中，建立一個抽籤遊戲。

但是 **不需要自己從零寫程式**。

本作業的重點是：

**學會用 AI 協助產生程式。**

---

## 五、使用 AI 產生程式的流程

以下是推薦流程。

### Step 1：描述需求

先清楚描述想要的功能。

例如可以問 AI：

```
請幫我做一個簡單的抽籤網頁
功能包含
- 可以輸入名單
- 每行一個名字
- 按下按鈕隨機抽出一個人
- 使用 HTML CSS JavaScript
- 適合初學者
```

AI 會產生一個完整範例。

---

### Step 2：閱讀 AI 產生的程式

不要直接貼上。

先理解：

* HTML 做什麼
* CSS 做什麼
* JavaScript 做什麼

至少要知道：

* 哪裡是按鈕
* 哪裡是輸入區
* 哪裡是結果顯示

---

### Step 3：放到 lab1

把 AI 產生的程式：

放到

```
lab1/index.html
```

如果有 CSS：

```
lab1/style.css
```

---

### Step 4：測試

直接在電腦開啟：

```
lab1/index.html
```

確認：

* 可以輸入名字
* 按下按鈕
* 會出現隨機結果

---

## 六、如何向 AI 改進功能

如果功能不夠，可以繼續問 AI。

例如：

```
請幫我把抽籤結果變得更醒目
```

或

```
幫我加一個重新抽籤按鈕
```

或

```
幫我把畫面變得更好看
```

AI 會提供修改版本。

---

## 七、修改完成後上傳 GitHub

回到 **GitHub Desktop**。

左側會看到修改過的檔案。

輸入 commit message，例如：

```
完成 lab1 抽籤遊戲
```

然後按：

```
Commit to main
```

再按：

```
Push origin
```

---

## 八、GitHub Pages 查看成果

如果 GitHub Pages 已經設定完成。

你的網站會在：

```
https://你的帳號.github.io/你的repo/
```

首頁會顯示：

* Lab1
* Lab2
* Lab3

點擊 **Lab1** 就能看到抽籤遊戲。

---

## 九、常見錯誤

### Lab1 打不開

請檢查：

* 是否有 `lab1/index.html`
* 連結是否正確

---

### GitHub Pages 沒更新

請檢查：

* 是否 commit
* 是否 push
* 等待 1~2 分鐘

---

### CSS 沒作用

請確認：

```
<link rel="stylesheet" href="style.css">
```

是否正確。
