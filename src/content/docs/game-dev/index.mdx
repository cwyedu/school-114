【課程成果】電腦遊戲製作：Game:bit 硬體互動與網頁系統整合

內容簡述

本計畫運用 Game:bit 遊戲擴充板 結合程式設計，成功復刻《超級瑪利歐》等三款經典遊戲，並開發 Web Showcase 網頁進行成果整合。過程中掌握了硬體按鍵事件處理、座標與碰撞運算，並解決了硬體控制與軟體邏輯銜接的技術難題。專案展現了資處科學生在硬體互動應用、程式邏輯化思考及網頁系統部署的綜合實力，呈現從硬體操控到雲端展示的完整開發歷程。

一、 學習背景與動機

在高職三年的資訊課程中，我一直對「軟硬體整合」如何創造互動體驗感到興趣。本學期在「電腦遊戲製作」課程中，我決定挑戰使用 Game:bit（基於 Micro:bit 的遊戲擴充模組） 來開發三款經典遊戲：《超級瑪利歐》、《小行星》與《太空侵略者》。

我希望透過實作掌握如何將硬體的實體按鍵（JoyStick 與 Buttons）轉化為遊戲中的動作指令，並學習物理模擬與物件導向架構。此外，身為資處科學生，我也練習利用 Vercel 網頁技術，將這些硬體開發的專案進行系統化展示。

二、 實際行動與技術應用

本專案核心技術在於硬體指令與軟體畫面的同步，實作流程如下：

硬體互動與核心邏輯：

Game:bit 硬體映射： 學習將 Game:bit 擴充板上的搖桿（X-Y 座標）與 A、B 按鍵，對應到遊戲角色的移動與跳躍射擊指令。

座標與碰撞系統： 在《超級瑪利歐》中，處理角色與硬體輸入之間的延遲（Latency），設計流暢的動態平台碰撞判斷。

陣列與效能優化： 在《太空侵略者》中，利用陣列管理大量敵機，確保在處理硬體中斷訊號時，遊戲畫面仍能維持穩定 FPS。

多媒體與硬體控制：

音效回饋： 利用 Game:bit 內建的蜂鳴器設計簡單的 8-bit 遊戲音效，提升遊戲的互動沉浸感。

視覺素材處理： 配合 Game:bit 的復古風格，自行剪輯與縮放像素素材，確保在不同解析度下的視覺一致性。

系統化網頁展示 (Web Showcase)：

為了讓專題能跨裝置展示，我開發了一個響應式網頁系統，將分散在硬體端的專案轉化為 Web 展示版本，並撰寫開發說明文件，提升作品的學術完整度。

三、 所培養的能力與專業知識

硬體控制與邏輯轉換： 學會處理實體訊號（Digital/Analog Signal）並將其數位化為程式邏輯，這是資處科結合物聯網（IoT）的基礎能力。

資料結構化管理： 學習將遊戲中的血量、分數等複雜數據進行結構化定義，確保程式具備高度的可讀性與擴展性。

雲端部署實務： 掌握從開發、測試到透過 Vercel 進行雲端部署的流程，具備初步的系統維運觀念。

四、 困難、反思與解決方式

遇到的困難： 使用 Game:bit 控制《超級瑪利歐》跳躍時，起初按鍵訊號處理不當，導致角色會出現「連跳」或「反應遲鈍」的問題。

解決方式： 我研究了「Debounce（防彈跳）」與「事件監聽優化」的觀念，重新設計按鍵偵測的頻率與邏輯，讓操作手感恢復經典遊戲的靈敏度。

反思： 我體會到硬體開發與純軟體開發最大的不同在於「物理限制」。這讓我學會了如何在受限的硬體資源下，透過程式優化（如精簡迴圈、減少不必要的運算）來達到最佳效能。

五、 未來連結與升學規劃

透過這次專題，我確定了自己對於「軟硬體開發」與「系統整合」的濃厚興趣。未來進入大學後，我希望能深入研究資訊管理或智慧應用領域，將資處科的資料整理專長，延伸應用到 AIoT 或互動系統開發上。這次從 Game:bit 硬體實作到 Web Showcase 網頁展示的完整經驗，已為我打下了跨領域技術整合的堅實基礎。
