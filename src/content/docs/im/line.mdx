## 生圖

我最近試了一下，用 ChatGPT 或是 Gemini，生成 4*5 一次 20 張是不錯的選擇，這樣就剛好兩次就可以完成 40 張了。

<Collapsible title="咒語">
```
任務目標： 從我提供的一張圖，創作一組 Q 版 LINE 貼圖。
佈局規格：

採用 4×5 網格排列（共20格）。
每個貼圖之間保有足夠間距。
背景必須為 純白色 (Pure White)，方便之後去背處理。
角色與風格一致性：

單一角色： 20格都必須是同一個角色。
角色設定： Q 版老師角色、銀框眼鏡、短髮、襯衫與識別證。
風格： 可愛 Q 版、向量插畫風格、粗黑色輪廓線、顏色鮮明、表情誇張。

文案與動作：
===========================這裡輸入你的內容=====
文字|動作
早安,拿咖啡微笑打招呼
午安,拿便當準備吃
===================================

輸出要求：
每格角色表情清楚、動作明確。
文字簡短、容易閱讀。
圖片邊緣乾淨，適合做 LINE 貼圖。
```

## 處理圖

我們需要將這些圖切割成單張，並且去背，最後調整成適合上架的規格。你可以用 Photoshop、Canva、Remove.bg 等工具來完成這些步驟。但是，就是這個但是，這些步驟都可以用 AI 來完成，甚至是一次完成的。不如就用 AI 寫個網頁的程式來完成。

[AI做的切圖去背](https://cwyedu.github.io/funs/line-image-cut/)

<Collapsible title="咒語">
```
Role:
你是一位精通電腦視覺 (Computer Vision) 與前端開發的資深專家。

Task:
撰寫一個單一 HTML 網頁工具，能自動偵測圖片中非白色背景的「獨立物件」，並將其提取為透明背景的 PNG。

Core Logic（自動偵測與精準去背）：

載入庫：
請使用 CDN 引入 opencv.js 與 jszip。

影像預處理：
將上傳圖片轉為灰階並進行高斯模糊 (Gaussian Blur) 以減少雜訊。
使用「二值化 (Thresholding)」配合 OTSU 演算法自動分離非白色背景區域。

【關鍵修正】
必須執行「形態學閉運算 (Morphological Closing)」，填補物件內部的小孔洞，確保生成的遮罩是實心的，避免誤刪物件內部的白色區域。

輪廓偵測 (Contour Detection)：
使用 cv.findContours 尋找獨立區塊。
僅保留面積前 20 大且寬高皆大於一定比例的物件，過濾微小雜訊。

【精準提取技術】：
禁止使用顏色過濾（Color-to-Alpha），因為這會導致物件內部的白色變透明。

必須使用「輪廓遮罩 (Masking)」：
為每個物件建立一個與偵測輪廓完全一致的單色遮罩 Mat，將原始區域映射至帶有 Alpha 通道的畫布上，僅將輪廓外的背景設為 Alpha 0（透明）。

處理與輸出：

強制尺寸：
將每個提取出的物件置中放入 370 px × 320 px 的畫布。

採用等比例縮放 (Contain)，確保物件不變形且不超出邊界。

釋放資源：
手動釋放所有產生的 cv.Mat 對象，防止瀏覽器記憶體洩漏。

打包下載：
將所有偵測到的物件打包為 ZIP 檔，格式為 PNG。

UI Requirements：

使用 Tailwind CSS 建立現代化介面。

畫布預覽區：
需即時用綠色框線 (Bounding Box) 標示出偵測到的 20 個物件位置。

提供「自動偵測並下載」按鈕，並在處理時顯示 Loading 狀態。

Output：
請提供完整且強韌的單一 HTML 原始碼。
```
</Collapsible>
